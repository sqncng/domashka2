Список контроля

Предназначены для группировки объектов по целям и периодичности контроля. Пользователь для своего удобства может создавать сколько угодно списков контроля.
На контроль могут быть поставлены "Технические проекты", "Задачи процесса", "Ошибки" и "Ветки".

Какие данные можно указать в списке контроля?

  • Наименование - краткое наименование списка контроля, которое позволит быстро находить его в списках и идентифицировать при постановке объекта на контроль.
  • Статус - может принимать одно из следующих значений: "Действует", "Не действует". Поставить объект на контроль можно только для списков в статусе "Действует".
  • Владелец - пользователь, который будет иметь возможность ставить объекты на контроль для данного списка.
  • Описание - подробное описание назначения списка контроля.
  • Информировать ответственных о контроле - флаг устанавливается, если владелец списка хочет, чтобы ответственные, чьи объекты поставлены на контроль в данном списке, знали, что их объект поставлен на контроль. При этом они будут видеть дату следующего контроля и смогут указывать результаты хода работ к ней.
