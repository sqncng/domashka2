Технический проект

Для управления проектированием и разработкой в СППР предназначен механизм технических проектов. При внесении изменений в имеющийся проект механизм технических проектов позволяет вносить эти изменения таким образом, чтобы они были увязаны с логической моделью, были прозрачны и информативны для других участников проекта.

Технический проект аккумулирует в себе всю информацию, касающуюся изменений - описание сути изменений, процессы, функции, профили пользователей, обработчики обновления, которые добавляются или изменяются в рамках технического проекта.

Форма технического проекта используется для предоставления возможности ведения информации о техническом проекте.

Приоритеты

Что указывается в полях Цели и Концепция?

В поле Цели указывается информация о том, ради чего ведется текущий технический проект. Информация о том, каким образом будут реализованы поставленные цели, указывается в поле Концепция.

Для чего может использоваться очередность?

Очередность проекта - это приоритет его выполнения. Данный показатель используется для ранжирования проектов в списках и отчетах.

Как организовать контроль за ходом выполнения технического проекта?

Для контроля работ в рамках технического проекта используются задачи процесса и шаблоны процесса. Шаблоны процесса позволяют определить перечень необходимых задач и этапов, взаимосвязи между ними и порядок их выполнения. В рамках шаблонов можно настроить перечень задач по разработке, контролю, согласованию и т.д.

Как связаны технический проект и входящие идеи?

Входящие идеи - идеи, которые планируется реализовать в рамках текущего технического проекта. Для описания того, как реализована идея, предназначено поле Описание реализации идеи.

Разработка технических проектов в ветках

Изменения в конфигурации могут быть произведены как в основной ветке проекта, так и в отдельной ветке, созданной для разработки технического проекта. Если разработка технического проекта ведется в отдельной ветке, то возможна загрузка изменений объектов метаданных из этой ветки. Сами изменения отражаются при этом в отдельном списке, в рамках которого возможно согласование изменений ответственными.

Документирование проектных решений

На закладке Проектные решения указывается информация о том, какие объекты системы затрагиваются в рамках выполения технического проекта. Это могут быть процессы, функции, разделы проекта, обработчики обновления, профили пользователей..

Как заполнить списки измененных в рамках технического проекта процессов, функций, разделов проекта, обработчиков обновления, профилей пользователей?

Заполнение измененных процессов, встроеных разделов проекта и добавленных обработчиков в обновления в рамках выполнения технического проекта производится только вручную с помощью кнопки Добавить.

Заполнение измененных или созданных сценариев работы пользователя, производится с помощью кнопки "Заполнить сценарии", либо вручную с помощью кнопки Добавить.

Заполение измененных функций производится либо вручную (с помощью кнопки Добавить), либо с помощью заполнения по измененных процессам (кнопка Заполнить - По процессам).

Заполение измененных профилей пользователей производится либо вручную (с помощью кнопки Добавить), либо с помощью заполнения по измененных объектам данных (кнопка Заполнить - По процессам и функциям).

Как зарегистрировать исходящую идею?

В процессе работы над техническими проектами возникают ситуации, когда появляются новые идеи. Если в рамках текущего технического проекта реализовать их невозможно, то регистрируются новые идеи.

Зарегистрировать идею на основании технического проекта возможно с помощью специальной команды в командном интерфейсе Зарегистрировать идею.

Как зарегистрировать ошибку в рамках текущего технического проекта?

В процессе разработки технических проектов возможно возникновение различных ошибкок реализации. Для этого предусмотрен механизм регистрации ошибок в рамках текущего технического проекта.

Зарегистрировать ошибку на текущий технический проект можно с помощью команды Зарегистрировать ошибку в командной панели формы. При этом в качестве ветки обнаружения ошибки используется ветка, в которой разрабатывается технический проект.

Для просмотра всего списка ошибок, зарегистрированных в рамках текущего технического проекта, предназначена гиперссылка в панели навигации формы Ошибки.

Для чего служит команда "Информация для хранилища"?

Команда Информация для хранилища позволяет на основании данных проекта формировать текст для закладки изменений в хранилище конфигурации и дальнейшей идентификации изменений в хранилище.

Для чего предназначены флаги индикации публикации и внесения изменений в справку и документацию технического проекта?

Флаги индикации публикации и внесения изменений в справку и документацию технического проекта предназначены для выполнения отбора технических проектов в списках и отчетах.

Для чего служит поле Каталог демонстрационной базы?

При работе над техническими проектами реализуется новый функционал программы, производятся изменения в существующем функционале. Для иллюстрации новых возможностей, наглядного ознакомления с ними пользователей системы полезно продемонстрировать эти возможности в информационной базе с использованием демонстрационных примеров. Базу можно разместить на общем ресурсе, доступном пользователям системы и для информации указать каталог размещения базы в техническом проекте.

Как используется команда Создать хранилище?

Если принимается решение о разработке технического проекта в отдельном хранилище, то необходимо выполнить создание этого хранилища. СППР позволяет создать такое хранилище автоматически на основании основного хранилища разработки. При этом в форме создания хранилища указываются необходимые настройки и запускается создание хранилища в фоновом режиме.

Ключевые операции

На закладке «Проектные решения», «Ключевые операции» доступен перечень ключевых операций, которые встраиваются или замеряются в рамках этого технического проекта.

Список ключевых операций содержит колонки:

  • Объект метаданных – объект метаданных, в модуль, форму или команду которого встраивается замер ключевой операции;
  • Ключевая операция – встраиваемая или замеряемая ключевая операция;
  • Фактическое время – длительность выполнения ключевой операции в секундах, полученное в результате тестирования.

После выбора объекта метаданных, в колонке «Ключевая операция», можно выбрать только ключевые операции, относящиеся к этому объекту метаданных. При изменении объекта метаданных или ключевой операции, фактическое время устанавливается в значение, равное нулю.

Строки в этой таблице могут иметь следующие цвета текста в зависимости от значения в колонке «Фактическое время»:

  • черный – фактическое время равно нулю (тестирование не проводилось);
  • зеленый – фактическое время меньше либо равно целевому времени;
  • красный – фактическое время превышает целевое время (требуется расследование и исправление).

Под списком ключевых операций расположено поле «Описание», которое связано с выбранной строкой в списке. В описании можно указать произвольный текст, который поможет выполнить замер длительности выполнения ключевой операции, например:

  • сценарий выполнения ключевой операции (что сделать, чтобы она выполнилась);
  • роли, под которыми следует ее выполнять, чтобы сделать замер с учетом RLS;
  • перечень объектов метаданных, которые она затрагивает при выполнении запросов и объем данных в них (максимальный, который может быть у пользователей);
  • произвольный комментарий, который может быть полезен разработчику, тестировщику или ответственному за производительность.

Дополнительные реквизиты

Для справочника технических проектов рекомендуется создать дополнительный реквизит для описания общих для всех ключевых операций технического проекта условий замеров.

В такой реквизит можно поместить следующую информацию:

  • расположение информационной базы, на которой следует выполнять замеры;
  • расположение обработки, которую следует использовать для наполнения тестовой информационной базы, если в техническом проекте разрабатываются новые объекты метаданных, и в тестовой базе для них нет данных.

Что показывате вкладка Результаты проверки?

На данной вкладке собираются ошибки по всем объектам затронутым техническим проектом (Процессам, Функциям, Разделам проекта и т.д.).

Приоритеты

В случае если в системе включено использование приоритетов и для проекта технического проекта корректно настроено ведение приоритетов, то в командной панели формы появляется подменю, которое содержит информацию о текущем приоритете и команды, позволяющие изменить приоритет.
