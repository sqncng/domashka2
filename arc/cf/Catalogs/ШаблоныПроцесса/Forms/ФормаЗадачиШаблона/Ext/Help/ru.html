Форма задачи шаблона

Указываемые в задаче шаблона значения реквизитов будут перенесены в соответствующие создаваемые по данному шаблону задачи.

  • Наименование - краткое описание задачи для идентификации в списках и отчетах. Обязательно для заполнения.
  • Исполнитель – если не указан, то будет заполняться по описанным в конфигурации правилам. Например, для технических проектов это будет ответственный за данный проект.
  • Контролирующий – если указан, то исполнитель не сможет самостоятельно отметить выполнение задачи. Такое право будет у контролирующего, которому исполнитель должен будет послать задачу на проверку.
  • Плановая длительность – планируемая длительность задачи в днях.
  • Описание – подробное описание задачи. Для задач, создаваемых по шаблону можно например, привести цитаты из регламента работы.
  • Вышестоящая задача – родительская задача текущей задачи.
  • Предшествующие задачи – задачи, выполнение которых требуется для начала работы над текущей. Например, задача «Подготовиться к открытию проекта» является предшествующей по отношению к задаче «Согласовать открытие проекта».
  • Планируемые ресурсы – для задач, для которых значение расходуемого ресурса установлено регламентом или заранее прогнозируемое. Например, для проведения выборочного аудита кода разработанного решения отпущено 2 часа.
  • Приоритет – Приоритет указывается относительно «приоритета по умолчанию» проекта. При создании задач по шаблону, будет учитываться приоритет технического проекта и приоритет, заданный в задаче шаблона процесса. Если шаблон процесса используется не для технического проекта, то во вновь создаваемых задачах будут указаны приоритеты, заданные в задачах шаблона.
