Шаблон процесса

Шаблоны процесса предназначены для описания типового процесса выполнения работ, по какой либо сущности.
В шаблоне описываются задачи, которые предстоит выполнить исполнителям, определяется назначение шаблона и согласуемые ресурсы.

Использование шаблонов процесса позволяет быстро создать необходимые задачи для выполнения работ по определенной сущности. В созданных задачах могут быть определены исполнители, плановое время выполнения и согласованы требуемые ресурсы.
Настройки шаблона позволяют определить контекст использования с точностью до предмета процесса и его владельца.

Для шаблона процесса доступно указание:


Статус:

  • Черновик – шаблон находится в стадии подготовки и не предназначен для использования.
  • Действует – действующий шаблон.
  • Не актуален – шаблон больше не используется.

Назначение:

  • Общий – шаблон доступен для создания задач по любому предмету.
  • Только для – требуется указание типа предмета, для которого предназначен шаблон. Например, для технических проектов.

Владелец шаблона: дополнительный признак, позволяющий ограничить контекст применения запроса. Например, можно указать проект, в рамках которого будет применяться шаблон работы над техническим проектом.

Описание: возможность развернуто написать, для каких целей предназначается шаблон.

Ответственный: пользователь, который отвечает за редактирование шаблона

Персональный: данный признак устанавливается в том случае, если ответственный за шаблон делал его «для себя» и не предполагал использования другими пользователями.

Задачи: список задач, которые будут созданы при использовании шаблона, их взаимосвязи, и предустановленные значения некоторых реквизитов создаваемых задач

Согласуемые ресурсы: указываются, какие ресурсы будут согласовываться при выполнении задач.

Например, типовой процесс выполнения технического проекта можно описать следующим образом. Ответственному за проект необходимо подготовиться к предстоящей контрольной точке. А затем согласовать прохождение контрольной точки. И так далее до прохождения последней контрольной точки и выполнения проекта.

Таким образом, шаблон данного процесса можно описать следующим образом. Последовательные задачи верхнего уровня: «Подготовиться к КТ 1» -> «Согласовать прохождение КТ 1» -> «Подготовиться к КТ 2» - …….«Согласовать прохождение КТ 6».

В качестве подчиненных задач к задачам типа «Согласовать КТ …» можно описать задачи для согласующих свои зоны ответственности на данной контрольной точке.

После создания задач по шаблону, ответственный за технический проект в подчиненных задачах типа «Подготовиться к КТ» будет самостоятельно детализировать требуемые действия для подготовки к согласованию контрольной точки.
