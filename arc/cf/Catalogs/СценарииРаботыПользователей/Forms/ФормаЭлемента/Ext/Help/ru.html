Сценарий работы пользователя

Сценарии работы пользователей (далее сценарии) используются для описания и выполнения действий пользователя с помощью механизма Автоматизированного тестирования платформы 1С:Предприятие.

Для написания сценариев используется Gherkin подобный язык.

Сценарий может быть создан в текстовом редакторе либо с помощью записи действий пользователя в клиенте тестирования.

Сценарий соотносится с функциями посредством привязки сценария к конкретным функциям.

Сценарий подчинен какому-либо проекту.

Сценарий привязан к функциональной модели.

Что указывается на закладке Описание?
  • Наименование. Обязательное поле. Наименование сценария должно быть уникальным в пределах проекта.
  • Проект. Обязательное поле.
  • Ответственный. Обязательное поле. При автоматической регистрации ошибок в СППР ошибка, полученная при запуске сценария пользователя может быть автоматически направлена ответственному за сценарий.
  • Функция. Обязательное поле. Привязка к функциональной модели.
  • Раздел проекта. Необязательное поле. При автоматической регистрации ошибок в СППР ошибка, полученная при запуске сценария пользователя может быть автоматически направлена ответственному за раздел проекта, указанный в сценарии.
  • Профиль пользователя. Необязательное поле. Если поле заполнено, то при компиляции сценария будет добавлена строка для запуска клиента тестирования под данным профилем пользователя. Логин, пароль и другие данные для запуска клиента тестирования указываются в форме "Настройки локального запуска сценариев" (команда "Настройка БД для запуска тестов").
  • Разрешено использовать в бизнес-процессах. Флаг выставляется, если сценарий планируется использовать в бизнес-процессах.
  • Разрешено использовать в других функциях. Если флаг отмечен, то данный сценарий можно будет вызвать из сценариев, привязанных к другим функциям того же проекта.
  • Описание. Текстовое описание сценария.

Редактор текста сценария.
  • Текст сценария можно написать в соответствующем поле.
  • Для автоформатирования текста сценария используется команда "Отформатировать".
  • Для подсветки в тексте сценария ключевых слов используется поле форматированного документа.
  • Команда "Сменить редактор сценария" используется, чтобы отключить раскраску текста сценария и перейти в режим редактирования в обычном текстовом документе.
  • Команда "Показать/скрыть список шагов" позволяет управлять видимостью панели, содержащей шаги сценариев.

Запуск сценария на выполнение.
  • Основной командной для запуска сценария на выполнение явялется команда "Запуск тестов".
  • Команда "Запуск тестов с текущего шага" позволяет выполнить сценарий с шага, на котором установлен курсор.
  • Команда "Запуск фреймворка тестирования с данным сценарием" позволяет открыть фреймворк тестирования с данным сценарием, но сценарий не будет запущен выполнение.
  • Команда "Настройка БД для запуска тестов" открывает форму настроек для локального запуска сценариев.
  • Команда "Сформировать тексты сценария для просмотра" компилирует сценарий и открывает его для просмотра.

Как найти сценарии, вызывающие данный?

Для этого используется команда "Найти в других сценариях".

Как создать на основании части сценария подсценарий?

Для этого используется команда "Преобразовать в подсценарий". Выделенные строки сценария будут преобразованы в подсценарий.
