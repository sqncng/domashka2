Тест

Для проверки наличия или отсутствия ошибок разрабатываемой информационной системы в СППР предусмотрен механизм тестов.

Элементы справочника тесты могут создаваться как вручную, так и при загрузке данных из системы контроля версий регламентным заданием Загрузка результатов тестирования.

Когда нужно заполнять ответственного?

Ответственного надо заполнить в том случае, если тест не связан с процессом или сценарием работы пользователя.
Ответственный будет использован для группировки в отчете Статус прохождения тестов в ветке.

Когда нужно указывать Имя в системе сборки?

Имя в системе сборки заполняется автоматически при работе регламентного задания Загрузка результатов тестирования.
Если тест был создан вручную, то Имя в системе сборки нужно заполнить в том случае, если планируется автоматически загружать результат выполнения теста из системы непрерывной интеграции.

Как связаны тест и настройки запуска сценариев пользователя и настройки запуска процессов?

Тесты могут быть разных типов, в том числе они могут быть связаны с существующими сценариями работы пользователя или процессами.
Настройка запуска сценария заполняется, если тест связан с запуском сценария работы пользователей.
Настройка запуска процесса заполняется, если тест связан с запуском процесса.

Как указать, что тест дополнительно связан с объектами метаданных?

Если тест связан с объектом метаданных, тогда надо установить флаг - Результаты теста детализируются по объектам метаданных. Это позволит выполнять детализацию результов выполнения теста до объектов метаданных в отчете Статус прохождения тестов в ветке.

Для чего нужно указывать раздел проекта?

Раздел проекта позволяет указать к какому разделу относится данный тест. Это используется для последующего анализа выполнения тестов в разрезе разделов проекта.
