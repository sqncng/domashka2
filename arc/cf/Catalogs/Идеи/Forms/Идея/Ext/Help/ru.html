Идея

Справочник "Идеи" служит для регистрации идей и пожеланий, выдвигаемых к информационной системе.

Идеи могут регистрироваться в произвольный момент времени и обрабатываться по мере необходимости. Например, при планировании функциональности очередной версии разрабатываемой конфигурации ряд идей может быть отобран для реализации в рамках данной версии.

Идеи регистрируются в рамках конкретного проекта. При регистрации идеи назначается ответственный за данную идею.

Идеи учитываются в разрезе источников, от которых они получены. Кроме того, идеи могут ранжироваться по степени важности.

Зарегистрированные идеи используются в качестве входящих идей для технических проектов. Идеи могут быть реализованы в рамках технических проектов..

Возможно ли при регистрации идеи приложить какую-либо дополнительную информацию, например, требовоания заказчика, описанные отдельным документом, содержащим схемы, изображения и т.п.?

Возможно ли при регистрации идеи приложить какую-либо дополнительную информацию, например, требовоания заказчика, описанные отдельным документом, содержащим схемы, изображения и т.п.?

Для соотнесения дополнительной информации с идеей предназначен механизм присоединения файлов. Этот механизм позволяет прикреплять в рамках текущей идеи файлы различных типов. Например, изображения, текстовые файлы, видеофайлы и т.п.

Как можно понять, в какие технические проекты включена та или иная идея?

Для того, чтобы увидеть список технических проектов, реализующих текущую идею, в панели навигации расположена ссылка Технические проекты, использующие идею, при переходе по которой будет открыт необходимый список.

Как заполнить источник идеи?

Идеи могут поступать из разных источников: от разработчиков, от пользователей, и т.д. Источники идей хранятся в справочнике Источники идей. Информация об источнике может использоваться для отбора идей.

Как используется статус идеи?

Статусы предназначены для отражения текущего состояния идеи. Статус может использоваться для отбора. Например, пользователь может отобрать только те идеи, которые были отклонены.

Статус идеи может принимать следующие значения:

  • Зарегистрирована - устанавливается автоматически при регистрации идеи,
  • Реализована - устанавливается в случае завершения реализации идеи,
  • Отклонена - устанавливается в случае отказа от реализации идеи.

Как используется тематика идеи?

Тематика идеи может использоваться для группировки по произвольным разделам.
Тематика может быть указана вручную в соответствующем поле, либо выбрана из выпадающего списка. В список включаются тематики, к которым ранее были отнесены другие идеи в рамках этого же проекта. Таким образом, можно группировать идеи, относя их к одной тематике.

Для чего используется флаг К просмотру (ответственным)?

Данный флаг автоматически включается при создании идеи и сигнализирует ответственному о том, что идею необходимо рассмотреть. После рассмотрения идеи ответственный может снять флаг. Таким образом, можно удобно отбирать идеи, которые еще не были рассмотрены.

Также возможно выполнение отбора по данному флагу.
