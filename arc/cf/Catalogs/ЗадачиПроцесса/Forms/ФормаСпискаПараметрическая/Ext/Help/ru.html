Что выводится в данную форму?

Отображается список задач по предмету, в который выведены следующие колонки:

  • Наименование
  • Состояние
  • Начало (плановая, если задача запланирована или фактическая, если задача выполняется или выполнена)
  • Окончание (фактическое, если задача выполнена или плановое если задача не выполнена)
  • Исполнитель
  • Направлена - кому направлено поручение по задаче.
  • Контролирующий
  • Колонки, отражающие ход согласования.

Какие быстры отборы предусмотрены в форме?
  • Состояние – по текущему состоянию задач. Доступен множественный отбор.
  • Исполнитель – по исполнителю задачи.
  • Направлена –по пользователю, которому направлено текущее поручение по задаче.
  • Зона соблюдения сроков – позволяет отобрать задачи с проблемами соблюдения плановых сроков. Доступен множественный выбор.

Настройки хода согласования.

В списке предусмотрено несколько колонок, которые отображают ход согласования ресурса. Некоторые из этих колонок уже выведены в список, остальные скрыты по умолчанию, но доступны для персонализации, используя команду «Изменить форму».

  • Согласовано – согласовано в текущей и дочерних задачах.
  • Согласовано (тек.) – согласовано в текущей задаче
  • Согласовано (доч.) – согласовано в дочерних задачах.
  • На согласовании – на согласовании в текущей и дочерних задачах
  • На согласовании (тек.) - на согласовании в текущей задаче.
  • На согласовании (доч.) – на согласовании в дочерней задаче.
  • Выполнено – запланировано в текущей и дочерней выполненных задачах.
  • % Выполнения – процент выполнения задачи, рассчитывается по плановым данным.
  • Выполнено (согл.) – согласовано в текущей и дочерних выполненных задачах
  • % Выполнения (согл.) – процент выполнения, рассчитывается по согласованным данным.

Для того чтобы в список выводились данные по ходу согласования, необходимо указать по какому виду ресурса или согласуемому ресурсу в данный момент необходимо отображать информацию. Сделать это можно в подменю "Еще" - "Ход согласования по..."

Доступные команды


Создание задач.

  • Параллельную – создает новую задачу, устанавливая родителя, такого же, как у задачи, на которой в данный момент в списке выполнено позиционирование в списке.
  • Подчиненную – создает новую задачу, для которой устанавливает в качестве родителя задачу, на которой в данный момент в списке выполнено позиционирование в списке.
  • Последующую – создает новую задачу, для которой устанавливает в качестве предшествующей задачу, на которой в данный момент выполнено позиционирование в списке.

Поручения по задачам.

  • Написать поручение – позволяет находясь в списке написать поручение по задаче. Открывается дополнительная форма, где можно указать адресата поручения и текст поручения.
  • Ответить – позволяет находясь в списке ответить на поручение по задаче. Открывается форма из предыдущей команды, но в ней уже заполнен адресат поручения автором текущего поручения по задаче.
  • Направить на согласование – позволяет быстро направить задачу согласующему ресурса. Команда доступна если в списке установлен отбор по ресурсу. Будет выполнена, если количество ресурса в статусе на согласовании больше нуля.
  • Направить на проверку – позволяет быстро направить задачу на проверку контролирующему задачи. Будет выполнена, если у задачи указан контролирующий и задача находится в статусе «Запланирована» или «Принята к выполнению».
  • Ознакомиться – позволяет быстро очистить поле "Направлена" в задаче. Будет выполнена, если задача направлена текущему пользователь.

Команды повышения статуса

  • Запланирована – позволяет находясь в списке повысить статус задачи до «Запланирована». Операция будет выполнена успешно, если задача находится в статусе «Отменена».
  • Принята к выполнению – позволяет находясь в списке повысить статус задачи до «Принята к выполнению». Операция будет выполнена успешно, если задача находится в статусе «Отменена» или «Запланирована».
  • Выполнена – позволяет находясь в списке повысить статус задачи до «Выполнена». Операция будет выполнена успешно, если задача находится в статусе «Отменена», «Запланирована» или «Принята к выполнению».

Также успех выполнения операции зависит от максимального возможного статуса каждой задачи и прав на изменение статуса в каждой задаче. Например пользователь не сможет перевести задачу в статус «Выполнена», если в ней указан контролирующий, и это другой пользователь

Согласование ресурсов

  • Согласовать – позволяет выполнить согласование ресурса, по которому установлен отбор в списке. Операция будет выполнена успешно если пользователь является согласующим ресурса в этой задаче, и есть что согласовывать. Команда не выполняет согласование дочерних задач. Данная операция доступна только в форме элемента задачи.
  • Частично согласовать – позволяет выполнить частичное согласование ресурса, по которому установлен отбор в списке. Будет открыта форма, в которой пользователь сможет указать, какое из запрашиваемого количества будет согласовано. Оставшееся количество будет записано в статусе «Не согласовано». Операция будет выполнена успешно если пользователь является согласующим ресурса в этой задаче, и есть что согласовывать. Данная команда может выполняться только для одной команды в списке.
  • Не согласовать – позволяет отклонить согласование ресурса , по которому установлен отбор в списке. Операция будет выполнена успешно если пользователь является согласующим ресурса в этой задаче, и есть что согласовывать. Команда не может быть выполнена для не согласования дочерних задач. Данная операция доступна только в форме элемента задачи

Если операция будет выполнена успешно, то задача будет перенаправлена исполнителю, в тексте поручения будут отражены результаты текущего согласования.

Создание задач по шаблону

Если для предмета и его владельца задан, хотя бы один шаблон, в командной панели списка будет отображаться группа создания задач по шаблону. В подменю будет выведен список всех доступных для данного предмета шаблонов.
После применения шаблона будут созданы определенные в нем задаче. Примененный шаблон в подменю будет помечен галочкой.
Если для предмета уже был применен шаблон. команды создания задач по шаблону будут недоступны. Для использования другого шаблона вначале потребуется удалить все созданные по другому шаблону задачи.
