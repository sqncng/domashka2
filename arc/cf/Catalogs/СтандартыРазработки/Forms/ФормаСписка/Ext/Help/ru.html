Стандарты разработки

Стандарты разработки - это набор требований и рекомендаций по разработке программного обеспечения. Поддержка стандартов в СППР позволяет:

  • ознакомиться с перечнем и текстами стандартов;
  • учитывать изучение и нарушение стандартов разработчиками;
  • отмечать, какие стандарты нарушены в ошибках;
  • автоматически синхронизировать перечень стандартов со внешней системой.

Список стандартов разработки является основным местом по работе с ними. Этот список можно использовать для:

  • ознакомления с полным перечнем стандартов и их текстами;
  • ознакомления с перечнем требующих изучения или новых стандартов при их открытии из формы Текущие дела;
  • ознакомления со статистикой изучения и нарушения стандартов (закладка Статистика);
  • управления процессом изучения стандартов.

Список стандартов

Список стандартов отображает все или часть стандартов (если установлены отборы) в иерархическом или плоском виде. Колонки таблицы содержат:

  • Стандарт - ссылка на стандарт или группу стандартов;
  • Состояние - состояние стандарта для выбранного пользователя (Новый, Требует изучения, Изучен);
  • Дата изменения - дата последнего изменения стандарта (стандарты, измененные в состоянии Изучен переходят в состояние Требует изучения);
  • Дата изучения - дата изучения или поручения изучения стандарта (по сути дата изменения состояния стандарта для пользователя);
  • Причина изучения - текстовое описание причины изучения;
  • Автор - пользователь, который изменил состояние стандарта.

"Причина изучения" имеет следующие особенности:

  • для себя пользователь может установить его сам при помощи команды Изучить;
  • для других его может установить пользователь с ролью Поручение изучения стандартов разработки при помощи команды Поручить изучение;
  • система сама может установить текст Стандарт изменен, если он был ранее изучен, а потом изменен.

Автором может быть:

  • сам пользователь, если он воспользоватлся командой Изучить;
  • другой пользователь с ролью Поручение изучения стандартов разработки", если он воспользовался командой Поручить изучение;
  • системный пользователь, если стандарт изменился в процессе синхронизации.

Состав списка стандартов можно изменять при помощи отбора:

  • Пользователь - пользователь, относительно которого просматривается список;
  • Изученные - отображать в списке стандарты в состоянии Изучен;
  • Новые - отображать в списке стандарты в состоянии Новый;
  • Требующие изучения - отображать в списке стандарты в состоянии Требует изучения.

Отборы Изученные, Новые, Требующие изучения показывают полный список в том случае, если они все установлены или сняты, в остальных случаях они отображают комбинацию по установленным значениям.

Команды

Команда Изучить позволяет установить выбранным стандартам состояние Требует изучения для себя. Можно рассматривать эту команду, как помещение в избранное для изучения.

Команда Подтвердить изучение позволяет установить выбранным стандартам состояние Изучен для себя. Подтверждая изучение выбранных стандартов, вы обязуетесь соблюдать их при разработке. Рекомендуется внимательно ознакомиться с выбранными стандартами, прежде чем подтверждать их изучение.

Команда Поручить изучение позволяет установить выбранным стандартам состояние Требует изучения списку указанных пользователей. Эта команда доступна пользователям с ролью Поручение изучения стандартов разработки.

Команда История изучения позволяет ознакомиться с хронологией изменения состояния выбранного стандарта для указанного пользователя.

Команда Режим просмотра вынесена из подменю Все для быстрого переключения между режимами просмотра. Удобно использовать, когда нужно найти несколько новых или требующих изучения стандартов, которые расположены где-то в иерархии стандартов.

Неявной командой открытия стандарта внутри СППР (форма стандарта разработки) или во внешней системе, является выбор из списка стандартов разработки (двойное нажатие мышкой или клавишей Enter).

Если необходимо в любом случае открыть форму стандарта разработки (а не перейти к просмотру стандарта во внешней системе), то необходимо начать его изменение (команда Изменить или клавиша F2).

Статистика

На закладке Статистика можно ознакомиться с количеством:

  • изученных стандартов (отображается индикатором и текстом);
  • нарушенных стандартов (нарушенные - это изученные, но не соблюденные);
  • несоблюденных стандартов (несоблюденные - это не изученные стандарты с которыми связаны ошибки);
  • требующих изучения стандартов;
  • новых стандартов.

Часть этой статистики (Требующие изучения и Новые) отображаются на форме Текущие дела, когда их количество больше нуля.
