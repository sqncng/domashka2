Функция системы

Функция – совокупность действий, выполняемых пользователями и внутренними механизмами проектируемой информационной системы (ИС) и связанных с преобразованием информации. Список функций ИС определяется в результате декомпозиции – разбиения функции на более простые составляющие (функции более низкого уровня).

Результатом проектирования функции являются:

  • схема функции,
  • описание функции,
  • описание объектов схемы,
  • декомпозиция функции на дочерние функции.

Иерархия функции определяется в дереве функций, после чего выполняется настройка связей функций.
Функции увязывают информацию о возможностях, поддерживаемых системой для реализации автоматизируемых процессов, потоках данных, обрабатываемых в информационной системе и исполнителях, исполняющих процессы в рамках конкретных функций.

Схемы функций строятся системой автоматически на основании иерархии и декларативного описания связей (указания входящей информации и исполнителей) функции.

В дереве функций присутствуют группы функций (функциональные блоки) и элементы (конечные функции). Группы функций могут включать другие группы и конечные функции. В процессе разработки фукнций может производиться перегруппировка функций с целью добиться нужной глубины и детальности модели. Для большей наглядности и удобства восприятия следует на одном уровне иерархии располагать не более 5-7 функций.

Конечные функции не подлежат детализации (разбиению на дочерние функции). Для конечной функции указываются ее связи (входы, управление) и исполнители.

Как использовать механизм проверки формальныхх правил проектирования?

В форме функции указывается общая информация о конечной функции, определяются ее связи и исполнители. На зкаладке "Схема функции" отображается схема, формируемая автоматически на основании информации о связях и исполнителях.

Что определяет тип функции, указываемый в поле "Тип"?

Тип функции позволяет отделить "обычные" функции, т.е. функции, выполняемые пользователем в процессе работы, от доугих функций- учетных механизмов и отчетов.

В зависимости от типа функции рри выборе исходящих объектов метаданных по умолчанию устаналивается фильтр по типу объектов: для "обычных" функций это справочники, документы и другие ссылочные объекты. Для учетных механизмов - это регистры, как правило, хранящие учетную информацию, генерируемую системой на основании первичных данных, указываемых пользователем. Для функций типа "отчет" указываются отчеты и обработки.

Функции различного типа отображаются на схеме разным цветом. Пояснения к цветовому оформлению функций в виде легенды приведены в форме функции под схемой.

Как используется флаг "Отображать в интерфейсе"?

Признак "Отображать в интерфейсе" позволяет соотнести функциональную модель с моделью интерфейса разрабатываемой конфигурации и разделить разрабатываемые функции по разделам командного интерфейса. При этом можно уточнить название соответствующего раздела в интерфейсе.

Что указывается в поле "Результат"?

В поле "Результат" указывается результат выполнения функции. Например, результатом функции "формирование заказов" будут являться сформированные заказы.

Результат выполнения функции может использоваться другой функцией. Например, результаты функций-учетных механизмов (информация о расчетах, остатках товаров и т.п.) могут использоваться другими функциями для анализа, заполнения документов и т.д.

Как заполнить связи функции?

Связи функции указываются на закладке "Входящие данные". В качестве связей выбираются другие функции, результат выполнения которых требуется использовать в текущей функции. При построчном вводе может использоваться подбор как по наименованию или коду функции, так и по тексту результата.

Для каждой связи следует указать вид связи - "вход" или "управление". Связи будут расположены на схеме с учетом их вида.

При разработке функциональной модели функции могут создаваться постепенно, разными ответственными. Поэтому, когда разработчик предполагает, что входом функции должна являться, например, информация о доступных товарах, функции, порождающей эту информацию в системе, может еще не существовать. В этом случае в таблице "Входящие данные" можно не указывать ссылку на функцию, породившую результат, а укзать только комментарий. Связь на схеме при этом отображается с текстом комментария и блок связи оформляется желтым цветом. В дальнейшем информация может буть уточнена и указана связь с функцией, порождающей нужный результат. Аналогично с исполнителями функции- если на момент разработки функции еще не определен нужный профиль, можно указать информацию об исполнителе в виде комментария, а впоследствии уточнить данные.

Что указывается на закладке "Исходящие объекты метаданных"?

На данной закладке указываются объекты метаданных, порождаемые в результате выполнения функции. Например, для функции "Оформление заказов" указывается документ "Заказ".

Как можно посмотреть более подробную информацию о связях функции?

Непосредственно на схеме функции можно выполнить двойной клик мышью на блоке интересующей связи. При этом откроется форма объекта, указанного в качестве связи.

Почему поля в форме могут быть недоступны для редактирования?

Объекты функциональной модели могут быть созданы на основании объектов другого проекта (источника). При этом для них может быть определено разное правило использования. Если в форме отображается информация об объекте источнике с гиперссылкой этого объекта и правилом использования Без изменений, то используемые поля такого объекта не подлежат редактированию. При необходимости можно изменить правило использования объекта, воспользовавшись гиперссылкой правила использования, после чего поля станут доступны для редактирования. При этом возможность изменения правила использования определяется правами пользователя в системе.
