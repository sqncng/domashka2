Версии проекта

В течение жизненного цикла проекта разрабатываемая в рамках проекта конфигурация постоянно развивается- при этом требуется разработка и выпуск новых версий проекта (конфигурации). В СППР для проекта ведется учет версий, данные версии используются при проектировании и разработке, исправлении ошибок, планировании и т.д.

Для версии следует указать ветку, в которой ведется разработка. Эта информация будет использоваться при загрузке метаданных и обработчиков обновления информационной базы. Также в разрезе веток ведется регистрация и обработка ошибок.

Информация о версии проекта редактируется в карточке версии.

Как соотносятся версии и сборки проекта?

В рамках одной версии проекта может быть создано неограниченное количество сборок, т.е. готовых вариантов версии проекта.
В форме для каждой версии в списке, расположенном в нижней части формы, отображаются сборки данной версии.

Почему некоторые версии могут не отображаться в списке версий?

Каждая версия имеет свой жизненный цикл. Для версии может быть указана Дата окончания поддержки. По умолчанию в списке не отображаются версии, поддержка которых прекращена, т.е. дата окончания поддержки для которых менее текущей даты.

Для того, чтобы отобразить полный список версий, необходимо в настройках списка снять флаг Отбор версий, поддержка которых не прекращена.
