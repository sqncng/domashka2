Версия проекта

В течение жизненного цикла проекта разрабатываемая в рамках проекта конфигурация постоянно развивается- при этом требуется разработка и выпуск новых версий проекта (конфигурации). В СППР для проекта ведется учет версий, данные версии используются при проектировании и разработке, исправлении ошибок, планировании и т.д.

Для версии следует указать ветку, в которой ведется разработка. Эта информация будет использоваться при загрузке метаданных и обработчиков обновления информационной базы. Также в разрезе веток ведется регистрация и обработка ошибок.

Что определяет порядковый номер версии?

Порядковый номер версии определяет номер версии в порядке следования версий друг за другом. При создании новой версии порядковый номер устанавливается автоматически на соновании имеющихся версий и может быть изменен вручную при необходимости.

Для чего используются дата начала и окончания разработки, предстоящей сборки версии?

Указанные даты предназначены, прежде всего, для информирования пользователей о сроках работ над версией.
Информация о предстоящей сборке отображается в ошибках, в которых версия фигурирует в качестве версии исправления, если исправление ошибки еще не включено в имеющиеся сборки.

Дата предстоящейй сборки является "скользящей", т.е. при завершении работ над очередной сборкой версии в данном поле следует указать дату следующей планируемой сборки.

Для чего используется срок включения библиотек в версию (поле Включение библиотек до)?

Срок включение библиотек определеяет, не позднее какой даты следует включить необходимые библиотеки в версию (если в проекте используются библиотеки).
Эта информация используется для контроля за ходом разработки и соотнесения планов разработки библиотек и проектов, использующих эти библиотеки.

Информацию о несоответствии сроков можно увидеть непосредстенно в форме, в таблице Версии включаемых библиотек. Например, если срок готовности библиотеки оказывается позже, чем срок включения библиотек в проект, будет отображена информация об этом. Для ответственных за планирование работ это является сигналом о необходимости согласования сроков.

Также соотнесение сроков разработки и выпуска библиотек и проектов, включающих библиотеки, может контролироваться при помощи отчетов Сводная информация по выпуску версий и Версии библиотеки в составе конфигураций.

Как заполняется поле Окончание поддержки командой Заполнить?

Дата окончания пооддержки по-разному заполняется для проектов, являющихся библиотеками, и для проектов, которые библиотеками не являются.

Для библиотек дата окончания поддержки рассчитывается с учетом информации о поддержке всех версий проектой, использующих данную версию библиотеки и устанавливается равной наиболее поздней дате окнчания поддержки использующей библиотеку версии.

Для обычного проекта (не являющегося библиотекой) дата окончания поддержки устанавливается равной дате выпуска версии, следующей за текущей. Т.е. предлагается осуществлять поддержку текущей версии до выпуска следующей.

Как заполняется информация в таблице Версии включаемых библиотек?

Список библиотек, включаемых в проект, заполняется системой автоматически на основании информации о включении библиотек в проект, указываемой в справочнике Проекты.

Для каждой библиотеки пользователь может указать конкретную версию библиотеки, которая включается в данную версию проекта.

Информация в остальных колонках таблицы выводится справочно и берется из соответствующих версий библиотек.

Встроенная сборка - номер сборки проекта-библиотеки. Если версия имеет отдельное хранилище и дата окончания поддержки версии еще на наступала, то загрузка сборок будет выполнена вместе с загрузкой метаданных проекта.
Последняя сборка - отображение текущей последней сборки проекта-библиотеки.
