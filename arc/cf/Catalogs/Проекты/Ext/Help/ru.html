Проекты

Проекты в СППР соответствуют разрабатываемым конфигурациям.
Большинство объектов информационной базы в СППР относится к тому или иному проекту. По проектам ведется разграничение доступа пользователей к информации, хранящейся в информационной базе.
Каждый проект включает основные настройки, касающиеся работы с информационной базой и относящиеся именно к данному проекту (правила обработки ошибок, настройки работы с метаданными, необходимость рассылки уведомлений и т.п.)

Для каждого проекта определяется, нужно ли вести список обработчиков обновления информационной базы. Эта функциональность доступна только для конфигураций, которые разрабатываются на основе Библиотеки стандартных подсистем.

Если обработчики обновления для проекта ведутся, то при загрузке обработчиков обновления в проекте автоматически обновляется таблица библиотек, которые поставляют обработчики обновления.

Для проектов в которых предусмотрена локализация в таблицу библиотек необходимо добавить библиотеку локализации и модуль локализации с соотвествующей отметкой (модуль локализации может быть только один). При генерации кода процедур обработки обновления для этого модуля будет формироваться отдельная процедура содержащая обработчики обновления локализованных объектов.
