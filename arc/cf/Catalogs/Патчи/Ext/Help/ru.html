Патчи

После исправления критичной ошибки часто возникает необходимость создать расширение для точечного исправления проблемы в конфигурации пользователя без необходимости обновления программы на новую версию. Для этой задачи подходит механизм патчей - по исправлению в хранилище создается небольшое расширение конфигурации, которое содержит в себе только измененный в рамках ошибки код.
Такое расширение содержит в себе информацию о сборках конфигурации, в которых оно может быть использовано и при обновлении пользователя на новую версию автоматически удаляется.

Новые патчи могут быть добавлены при установке в карточке ошибки флага Создать патч по исправлению ошибки или же независимо, по кнопке Создать в списке патчей.

Ограничения создание патчей:

  • автоматическое создание патчей возможно только если при разработке конфигурации используется хранилище Git
  • при автоматическом создании в патч помещаются только изменения кода, изменения форм пропускаются, но их можно перенести в собранный патч вручную
  • запрещается вносить изменения данных - добавлять новые справочники, документы, регистры и т.д., а так же расширять состав реквизитов таких объектов

В панели Текущие дела на рабочем столе можно настроить уведомления о необходимости проверки созданного патча и его публикации.
Уведомления о проверке отправляются автору исправления ошибки или автору патча, если он был создан не на основании зарегистрированной в СППР ошибке. Уведомления о публикации отправляются разработчику, проверяющему и аудитору, если для проекта включен дополнительный контроль перед публикацией при наличии у них прав на публикацию патчей. Если у них прав на публикацию нет, то уведомления отправляются только ответственным за публикацию, которые указываются на закладке Патчи в карточке проекта.
Уведомления в текущих делах можно настроить, включив или отключив вывод в Центр оповещения.
