Правило проверки объектов

Правила проверки являются частью механизма аудита модели разрабатываемой системы на соответствие формальным правилам проектирования. Система содержит предустановленный набор правил, предлагаемый для использования. Данный набор может быть расширен или изменен по усмотрению ответственного за корректность модели информационной системы. Данный механизм позволяет выявить ошибки проектирования и зафиксировать результаты проверки. Для анализа результатов проверки используется отчет Результаты проверки объектов.

Проверка соответствия проекта установленным правилам проверки выполняется автоматически при помощи соответствующего регламентного задания. Если в результате проверки выявляются ошибки, информация об этих ошибках записывается в регистр сведений Результаты проверки объектов.

Как используется поле "Важность"?

Поле Важность используется для ранжирования ошибок, выявляенных в ходе отработки правила проверки.

Что означает флаг "Предустановленное"?

Флаг Предустановленное означает, что данное правило поставляется в составе конфигурации и является рекомендуемым для использования.

Что означает флаг "Активно"?

Флаг Активно означает, что при выполнении регламентного задания, выполняещего проверку проекта на соответствие правилам проверки, данное правило будет учтено и при обнаружении ошибок они будут зафиксированы. Если флаг Активно не установлен, правило будет игнорироваться при выполнении автоматической проверки.

Как можно выполнить проверку правила?

Выполнить проверку конкретного правила можно, воспользовавшись командой Применить правило.

Как использовать текст запроса?

Механизм применения правил проверки осуществляет выполнение запросов в базе с заранее подготовленным текстом запроса, в результате чего выбираются данные, имеющие ошибки.
Соответственно текстз запроса должен быть составлен иаким образом, чтобы выбрать данные, удовлетворяющие определенным критериям.

Например, если ошибкой считается наличие в табличной части ссылок на объекты, помеченные на удаление, запрос должен выбрать объекты нужного вида, содержащие в табличной части ссылки на объекты, помеченные на удаление.

При создании нового правила в качестве текста запроса формируется комментарий, описывающий подробные требования к полям текста запроса.

Как можно посмотреть результаты проверки данного правила?

Для того, чтобы посмотреть результаты проверки правила, можно воспользоваться командой Результаты проверки в панели навигации формы.

Совет. При разработке правил проверки можно воспользоваться командой Показать результат. Выполнение данной команды не ведет к формированию записей об ошибках, а формирует отчет, содержащий информацию об ошибках и позволяющий оценить результаты выполнения проверки по данному правилу.
