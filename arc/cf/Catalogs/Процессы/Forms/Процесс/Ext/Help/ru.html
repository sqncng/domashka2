Процесс

Процессы используются для описания реальных процессов предприятия, которые необходимо автоматизировать. Процессы описываются с необходимой степенью детализации, при этом детализация выполняется с помощью шагов процесса.

Фактически описания процессов в СППР являются описанием сценария действий пользователя при реализации тех или иных его должностных обязанностей.

Процессы соотносятся с функциями посредством привязки шагов процесса к конкретным функциям. Таким образом обеспечивается возможность реализации автоматизируемого процесса в рамках конкретной функциональности.

Для организации тестирования процессов их нужно соотнести со сценариями пользователей. Это делается посредством привязки шагов процесса к конкретным сценариям пользователей. Таким образом, обеспечивается возможность запуска автоматизируемого процесса на выполнение с помощью платформенного механизма автоматизированного тестирования.

Запуск процесса на выполнение
  • Основной командой для запуска процесса на выполнение является команда "Запуск тестов".
  • Команда "Запуск фреймворка тестирования с данным сценарием" позволяет скомпилировать процесс в исполняемый сценарий и открыть фреймворк тестирования для отладки сценария.
  • Команда "Настройка БД для запуска тестов" открывает форму настроек для локального запуска скомпилированных сценариев и процессов.
  • Команда "Сформировать тексты сценария для просмотра" компилирует процесс и открывает его для просмотра в СППР.

Параметры процесса

На вкладке "Параметры процесса" указываются параметры процесса, которые являются общими для всех шагов процесса и определяют его поведение, когда процесс запускается на выполнение.
Например, таким параметром может быть "Организация", "Контрагент", "Номенклатура" и т.д.
Создание таких параметров процесса позволяет сделать из независимыми от данных в конкретной базе данных и сделать процессы более универсальными.
