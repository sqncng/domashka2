Запуск тестирования

В процессе разработки информационной системы может понадобиться регистрация событий запуска сборки/тестирования, возникающих в системе непрерывной интеграции.
Для учета этих событий предусмотрен механизм работы с документом Запуск тестирования.

Запуск тестирования может вводиться вручную или автоматически, при загрузке данных из системы непрерывной интеграции регламентным заданием Загрузка результатов тестирования.

Как указывается связь запуска тестирования с событием в системе непрерывной интеграции?

Для этого в поле Pipeline id нужно указать идентификатор запуска сборки в системе непрерывной интеграции.
Также в поле Тег указывается тег, установленный сборке в системе непрерывной интеграции для данной сборки.

Как указывается связь запуска тестирования с системой управления версиями?

Для этого в поле Имя ветки в git указывается имя ветки из системы управления версиями, для которой была запущена эта сборка.

Когда заполнятеся поле Проект?

Поле Проект является обязательным. Оно определяет, к какому проекту относится данный запуск тестирования.

Когда заполнятеся поле Сборка версии?

Поле Сборка версии заполняется в том случае, если данный запуск тестирования привязан к конкретной сборке проекта.

Что означает флаг Данные загружены?

Флаг Данные загружены означает, что данные по данной сборке из системы непрерывной интеграции загружены полностью.
Т.к. тесты могут выполняться длительное время, то регламентное задание Загрузка результатов тестирования может загружать данные частями. Т.е. результаты выполнения тестов, которые уже успели завершиться, будут загружены в СППР и будут доступны для анализа. Результаты выполнения оставшихся тестов будут загружены при следующем запуске регламентного задания Загрузка результатов тестирования.
