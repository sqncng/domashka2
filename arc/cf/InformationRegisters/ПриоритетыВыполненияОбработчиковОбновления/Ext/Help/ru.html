Приоритеты выполнения обработчиков обновления

Хранит результаты решения проблем по конфликтам обработчиков обновления.

Записи добавляются автоматически при записи обработчиков обновления. Для отложенных обработчиков обновления проблемами считаются любые пересечения по читаемым и изменяемым данным. Для монопольных - только повторная запись объектов.

При добавлении записи в регистр, проблеме присваивается статус Необходим анализ. Вручную этот статус нужно поменять на один из окончательных. Для монопольных обработчиков и отложенных список статусов разный. При изменении статуса для одног обработчика автоматически меняется статус и для другого.

Для монопольных:

  • Нужно переделать обработчики - т.е. избавиться от повторной записи
  • Нет проблемы - при выборе этого пункта необходимо указать причину допустимости повторной записи)


Для отложенных обработчиков результат решения проблемы влияет порядок помещения в очередь обработчиков:

  • Необходим анализ - необходимо принять решение по последовательности исполнения обработчиков. Пока есть обработчики, помеченные таким образом, построение очереди отложенных обработчиков невозможно.
  • Нужно переделать обработчики - т.е. избавиться от повторной записи или зависимости по читаемым/изменяемым данным. Пока есть обработчики, помеченные таким образом, построение очереди отложенных обработчиков невозможно.
  • Все равно в какой последовательности - при построении очереди обработчики могут быть размещены в любой последовательности. При это если между обработчиками есть конфликт повторной записи объектов, они гарантированно будут размещены в разных очередях.
  • Сначала обработчик 1 - при построении очереди сначала будет поставлен обработчик 1
  • Сначала обработчик 2 - при построении очереди сначала будет поставлен обработчик 2
