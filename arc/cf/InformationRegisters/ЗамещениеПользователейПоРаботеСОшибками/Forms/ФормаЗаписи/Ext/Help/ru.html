Замещение пользователя по работе с ошибками

Информация о замещении пользователей используется при работе с ошибками. Механизм работы с ошибками в ряде случаев автоматически направляет ошибку тому или иному пользователю. Например, отработанную ошибку предлагается направить для закрытия пользователю в соответствии с настройками работы с ошибками. Если пользователь временно отсутствует, либо это какой-то служебный пользователь (если ошибка регистрируется при помощи внешних систем с использованием веб-сервиса), настройки замещения позволяют организовать автоматическое перенаправление ошибки другому пользователю в соответствии с указанными настройками.

В форме указываются настройки замещения пользователя. Если проект указан, то настройка распространяется только на работу с ошибками по указанному проекту. Если проект не указан - настройка действует для всех проектов. При этом одновременно могут быть заданы несколько настроек для одного и того же замещаемого пользователя. Например, по двум указанным проектам могут быть назначены какие-либо замещаюшие пользователи и дополнительно может быть заведена настройка без указания проекта, которая будет распространяться на остальные проекты.

Замещающий пользователь может не указываться. В этом случае любой текущий пользователь, работающий с ошибкой, может выступать в роли замещающего пользователя и самостоятельно обрабатывать ошибки.
