В чем различие между индивидуальными уведомлениями пользователя и уведомлениями по техническим проектам?

Индивидуальные уведомления формируются для пользователя по адресному принципу. Например, если пользователь является исполнителем задачи, то при направлении ему задачи уведомление будет сформировано для пользователя как исполнителя.

Уведомления по техническим проектам действуют по принципу подписки. Если пользователь "подписался" на получение уведомлений по техническим проектам, относящимся к тому или иному проекту, то уведомления для него будут формироваться независимо от того, является ли он участником технического проекта или нет.

Для чего служит флаг Использовать локальные копии ИБ для воспроизведения ошибок?

Для удобства работы с ошибками СППР позволяет организовать хранение информационных баз 1С:Предприятия, требуемых для воспроизведения ошибок, на едином сетевом ресурсе, доступном всем пользователям. При эжтом информационные базы могут быть открыты непосредственно из ошибки.

У пользователя есть возможность открывать информационнеы базы как напрямую с сетевого ресурса, либо скопировать их локально средствами СППР и открывать локальную копию.

Установка флага означает, что по умолчанию используются локальные копии ИБ. Однако в каждом конкретном случае пользователь может открыть и общую базу.

Что требуется указать в поле Каталог программы для работы с ошибками?

В данном поле требуется указать путь к исполняемым файла 1С:Предприятия, используемым для работы с информационными базами, прикладываемыми к ошибкам.

Что следует указывать в поле Расположение локальной копии базы?

В поле следует указать каталог, в который будут помещаться информационные базы при их отркытии из ошибок при условии использований локальных копий баз.

Какой способ создания локальных копий ИБ при работе с ошибками следует выбрать?
  • В варианте Создавать каталог по коду ошибки система в каталоге расположения локальных копий баз для каждой ошибки создает подчиненный каталог, имя которого совпадает с кодом ошибка и в него помещает копию базы.
  • В варианте Использовать один каталог базы копируются в один каталог, при этом предыдущая копия базы, расположенная в каталоге, удаляется.

Соответственно, пользователь может выбрать наиболее подходящий ему вариант.
