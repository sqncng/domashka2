Загрузка метаданных

Загрузка метаданных позволяет загрузить информацию об объектах метаданных, подсистемах разрабатываемого проекта, библиотеках и версиях библиотек, а так же обработчиках обновления информационной базы непосредственно из разрабатываемой конфигурации в СППР. Метаданные требуются для взаимосвязи логической и физической моделей при проектировании. В процессе разработки, изменения метаданных и обработчиков обновления указываются в технических проектах в качестве одной из составляющих проектных решений. Метаданные могут указываться в ошибках. Информация загружается и хранится в справочниках Проекты, Объекты метаданных, Подсистемы и Обработчики обновления информационной базы.

Загрузка может производиться

  • из основного хранилища конфигурации. При этом, в зависимости от настроек могут загружаться или только объекты метаданных, или только обработчики обновления совместно с информацией о подсистемах, библиотеках и версиях библиотек, или же и те, и другие данные.
  • из хранилищ версий, для которых указано, что они разрабатываются в отдельном хранилище. При этом загружаются только обработчики обновления и информация о подсистемах и библиотеках и версиях библиотек.
  • из хранилищ технических проектов. При этом загружаются только объекты метаданных, но не все, а те, которые отсутствуют в основном хранилище

Загрузка из основного хранилища и из хранилищ поддерживаемых версий может производиться регламентно по расписанию, настраиваемому в карточке проекта.

При загрузке обработчиков обновления им присваивается версия по хранилищу, из которого производилась загрузка.
Осуществляется заполнение информации о подсистемах проекта и библиотеках стандартных подсистем. В справочнике Проекты заполняются сведения о режимах, в которых выполняются отложенные обработчики обновления, включенные в библиотеки и информация о версии, с которой осуществляется поддержка параллельного режима выполнении отложенных обработчиков.
В коде описания обработчиков может содержаться информация, читаемых, изменяемых и блокируемых данных, а также о взаимном порядке выполнения обработчиков. Эта информация загружается по умолчанию для обработчиковк, отнесенным к подсистемам, которые не разрабатываются в текущем проекте, т.к. для разрабатываемых в текущем проекте обработчиков информация, наборот, заполняется в СППР и выгружается в код. Но есть опциональная возможность загрузить информацию и по обработчикам разрабатываемых бибилиотек.

Обработка загрузки метаданных может использоваться для загрузки метаданных как из основного хранилища разработки, так ииз хранилищ технических проектов. В зависисмости от этого различается состав полей, достунпых в обработке. При загрузке метаданных из основного хранилища требуется указать проект, метаданные которого следует загружать. При загрузке в рамках технического проекта проект определяется по данным технического проекта автоматически.

Как указать источник для загрузки объектов метаданных?

Для загрузки метаданных из основного хранилища настройки подключения к информационной базе и хранилищу задаются в проекте. При загрузке используются эти настройки - получаются системой автоматически из проекта.

Для чего используется поле Версия платформы и какие версии доступны для выбора из списка в данном поле?

Для загрузки метаданных используется пакетная выгрузка конфигурации в файл при помощи конфигуратора, запускаемого программно средствами СППР. Если конфигурация обновляется из хранилища и для разработки используется сервер хранилищ, то версия конфигуратора должна соответствовать версии сервера хранилищ.

Выгрузка конфигурации в пакетном режиме и загрузка метаданных в СППР происходит на сервере. Поэтому соответствующая версия конфигуратора должна быть установлена на сервере. Пользователь, от имени которого работает сервер 1С:Предприятия, должен иметь права на работу с информационной базой, используемой для выгрузки метаданных.

В списке выбора доступны те версии платформы, которые установлены на сервере.

Как можно управлять флагом Обновить конфигурацию из хранилища перед загрузкой?

В настройках проекта администратором задается настройка Всегда обновлять конфигурацию ИБ из хранилища при загрузке метаданных. Если данная настройка включена, то при загрузке метаданных отключить принудительное обновление конфигурации из хранилища может только пользователь, обладающий полными правами, или администратор проекта.

Чем отличаются варианты загрузки из Базы тех. проекта и Хранилища при загрузке метаданных в рамках технического проекта?

Загрузка из базы технического проекта подразумевает сценарий, когда имеется база разработки, подключенная к хранилищу технического проекта. Для загрузки необходимо указать путь к данной базе, имя пользователя базы и его пароль. При необходимости базу можно обновить из хранилища технического проекта, установив флаг Обновить конфигурацию из хранилища перед загрузкой. Адрес хранилища для обновления при этом используется тот, который указан в техническом проекте. Для подклоючения к хранилищу требуется указать имя пользователя хранилища и его пароль.

При варианте загрузки из Хранилища не требуется наличие информационной базы, из которой будут загружаться метаданные. В данном случае информационная база будет создана автоматически на основании конфигурации хранилища средствами СППР. Достаточно указать только имя пользователя хранилища и его пароль. Адрес хранилища берется из технического проекта.
