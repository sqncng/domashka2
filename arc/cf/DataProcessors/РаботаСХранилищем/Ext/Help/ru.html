Работа с хранилищем

Помощник в различных режимах запуска позволяет:
- автоматизировать перенос изменений из хранилища технического проекта в основное хранилище;
- автоматизировать процесс обновления описания обработчиков обноваления информационной базы;
- захватывать объекты в хранилище версии.

Актуализация хранилища технического проекта

На этом шаге необходимо обеспечить, чтобы конфигурация технического проекта отличалась от конфигурации основного хранилища только на изменения, внесенные техническим проектом.

Для этого нужно выполнить следующие шаги:

  • Захватить измененные объекты в основном хранилище
  • Создать файл поставки с помощью механизма обновления
  • Сравнить и объединить конфигурации

Если процесс актуализации выполняется достаточно долго, то рекомендуется делать это в несколько проходов: на первом проходе - без захвата объектов, на втором проходе - с захватом объектов.

Перенос изменений в хранилище версии

На этом шаге необходимо перенести изменения и поместить их в хранилище.

Для этого нужно выполнить следующие шаги:

  • Сохранить конфигурацию технического проекта в файл
  • С помощью сравнения-объединения перенести изменения и поместить объекты в хранилище

Обновление описания обработчиков обновления в хранилище версии

Шаг присутствует только тогда, когда технический проект содержит обработчики обновления. На этом шаге перестраивается очередь отложенных обработчиков обновления, формируется текст модулей описания обработчиков обновления, модули помещаются в хранилище версии, а также происходит увеличение последней цифры номера версии.

Захват объектов в хранилище версии

Особый режим запуска обработки из формы ошибки или глобального командного интерфейса (Разработка-Сервис).
Отображается список захватываемых объектов. Способы наполнения списка
- при вызове из формы ошибки переносятся упомянутые в ошибке объекты;
- заполнить по результатам глобального поиска в конфигураторе;
- заполнить по результатам поиска по ссылкам в конфигураторе;
- выбрать объекты в дереве метаданных.
