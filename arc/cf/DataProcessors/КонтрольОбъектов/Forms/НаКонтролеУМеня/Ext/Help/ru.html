Для чего предназначена эта форма?

Это рабочее место, которое позволяет осуществлять контроль объектов, поставленных на контроль пользователем в выбранном списке. В нем можно отражать результаты и решения, просматривать историю и снимать объекты с контроля.

С чего начать работу?

Выберите список контроля. Список заполнится данными об объектах, контроль которых сегодня еще не выполнялся.

Что отображается в списке?
  • Объект контроля - объект, который пользователь поставил на контроль для выбранного списка контроля.
  • Ответственный - ответственный за выполнение работ, может отличаться от ответственного в самом объекте контроля.
  • Статус - выводится информация о текущем статусе объекта контроля и дате перехода на этот статус, там где это возможно.
  • Решение - текст решения, которое было принято при предыдущем контроле объекта и дата этого решения.
  • Закончить к.. - ожидаемая дата окончания завершения работ, определенная при предыдущем контроле объекта.
  • Результат - результат и дата его указания. Может быть указан как контролирующим, так и ответственным за объект контроля, в случае если в настройках списка контроля.

Как рекомендуется организовывать работу со списком?
  • Нужно определить и указать в элементе справочника "Список контроля" плановую дату объектов по данному списку.
  • В определенный для контроля день выбрать список контроля, он будет заполнен объектами, контроль которых был выполнен ранее сегодняшнего дня.
  • При указании решения по объекту или снятии его с контроля он будет автоматически пропадать из списка.
  • Когда список опустеет надо задать следующую дату контроля.

В моем списке пусто, я не ставил объекты на контроль. Как это сделать?

На контроль можно поставить при помощи команд расположенных в форме списка или элемента для следующих объектов:

  • Технические проекты
  • Задачи процесса
  • Ошибки
  • Ветки

Как сделать отметку о выполненном контроле объекта?

Для этого предназначена команда "Отразить результаты и решения". С помощью неё можно:

  • Указать результат на текущую дату.
  • Указать решение, принятое по результатам контроля.
  • Изменить ответственного.
  • Изменить ожидаемую дату окончания.

Команда может быть выполнена как для одного, так и для нескольких контролируемых объектов.
При этом, если команда выполняется для одного объекта то:

  • Возможно редактирование ранее указанного результата.
  • Поле решение предзаполняется предыдущим решением.

Если команда выполняется для нескольких объектов, то действуют следующие ограничения:

  • Недоступно редактирование ранее введенного результата. Он будет заменен на новый указанный результат.
  • Недоступно использование предыдущего решения как основы для нового решения. Его нужно будет вносить с "чистого листа"
  • Для смены ответственного или ожидаемой даты выполнения нужно будет явно указать намерение сделать это, установив соответствующие флаги.

Возможно ли изменить решение, и как найти запись с объектом контроля, ведь она пропала из списка?

Для того чтобы найти запись с объектом контроля, по которому уже было внесено решение, необходимо изменить дату отбора "Не отработанные на" с текущей даты на следующий день.
Отредактировать ранее внесенное решение нельзя, но можно указать новое решение.
В течении дня можно указывать сколько угодно решений, все они будут сохранены в истории контроля.

Возможно ли указание результатов без указания решения?

Да возможно, соответствующая команда находится в контекстном меню списка.
Может быть выполнена как для одного, так и для нескольких объектов сразу.

Как снять объект с контроля?

Выполнить соответствующую команду, расположенную в командной панели и контекстном меню списка

Как посмотреть историю контроля?

Выполнить соответствующую команду, расположенную в командной панели и контекстном меню списка

Отражены решения по всем объектам, список пуст. Что-то еще нужно сделать?

Нужно установить дату следующего контроля. Сделать это можно при помощи команды изменить, расположенной у выведенной в шапку информации о текущей плановой дате контроля.
